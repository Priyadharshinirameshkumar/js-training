## Why npm Scripts are Useful

npm scripts provide simple and consistent commands to run a project.

Instead of remembering long commands, team members can use the same script names such as `npm start`, `npm test`, or `npm run dev`.

If the command changes in the future, only the script inside `package.json` needs to be updated, and everyone on the team continues using the same npm command.

## dependencies vs devDependencies

### dependencies

These packages are required for the application to run in production.

Examples:
- dayjs
- express
- mongoose

### devDependencies

These packages are only needed during development.

Examples:
- nodemon
- eslint
- prettier

nodemon belongs in devDependencies because it automatically restarts the application during development. It is not required when the application is deployed to production.

## package.json vs package-lock.json

### package.json
- Contains project information.
- Lists project dependencies.
- Stores npm scripts.
- Can be edited manually.

### package-lock.json
- Automatically generated by npm.
- Stores the exact versions of installed packages.
- Ensures every developer installs the same package versions.
- Helps maintain consistency across different systems.

## npm install vs npm ci

### npm install

- Installs all required packages.
- Updates package-lock.json if needed.
- Used during normal development when adding or updating packages.

### npm ci

- Installs packages exactly as listed in package-lock.json.
- Deletes the existing node_modules folder before installation.
- Faster than npm install.
- Used mainly in Continuous Integration (CI) and deployment environments to ensure consistent installations.
